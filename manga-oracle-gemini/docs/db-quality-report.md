## 優先順位

1. タイトル重複候補を確認する
   - 表記ゆれ、続編、完全な重複を切り分ける。
   - 完全重複なら削除または別作品への差し替えを検討する。
2. 巻数未確認 `0` の扱いを決める
   - Webtoonや連載中作品は、巻数不明として `0` を許容する運用でもよい。
   - 正しい単行本巻数が分かる作品は更新する。
3. タグ品質を見直す
   - 推薦精度に直結するため、人気作、SEO記事掲載作、テーマ記事掲載作から優先して整える。

## コマンド

概要:

```bash
npm run check:db
```

詳細:

```bash
npm run check:db -- --details
```

厳格チェック:

```bash
npm run check:db:strict
```

## 注意

DBは診断結果の品質に直接影響するため、機械的な一括修正よりも、重複IDや人気作品など影響の大きい部分から少しずつ見直す方が安全です。
