## 優先順位

1. タイトル重複候補を確認する
   - 表記揺れ、続編、完全な重複を分ける。
   - 完全重複なら片方を削除または別作品に差し替える。

2. 巻数 `0` の扱いを決める
   - 不明巻数として許容するなら、検査ルール側を `0` 許容に変更する。
   - 正しい巻数が分かる作品は更新する。

3. タグ品質を見直す
   - 推薦精度に直結するため、人気作品・SEO記事掲載作品から優先して整える。

## コマンド

概要:

```bash
npm run check:db
```

詳細:

```bash
npm run check:db -- --details
```

厳格チェック:

```bash
npm run check:db:strict
```

## 注意

DBは推薦精度に直接影響するため、機械的な一括修正よりも、重複IDと人気作品から小さく直す方が安全です。
